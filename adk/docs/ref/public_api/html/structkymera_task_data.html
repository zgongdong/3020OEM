Detailed Description

Kymera instance structure.

This structure contains all the information for Kymera audio chains.

Field Documentation

uint8 a2dp_seid

The current A2DP stream endpoint identifier.

kymera_chain_handle_t chain_input_handle

The input chain is used in TWS master and slave roles for A2DP streaming and is typified by containing a decoder.

kymera_chain_handle_t chain_tone_handle

The tone chain is used when a tone is played.

union { ... } chainu

The output_vol_handle/sco_handle chain are used mutually exclusively. The output_vol_handle/sco_handle both contain OPR_SOURCE_SYNC/OPR_VOLUME_CONTROL. These chains are unioned to simplify volume control code: output_vol_handle can always be used with OPR_VOLUME_CONTROL regardless of whether SCO/A2DP is active.

uint16 lock

A lock bitfield. Internal messages are typically sent conditionally on this lock meaning events are queued until the lock is cleared.

micSelection mic

Which microphone to use during mic forwarding

uint32 output_rate

The current output sample rate.

kymera_chain_handle_t output_vol_handle

Used in TWS master and slave roles for A2DP streaming

Transform packetiser

The TWS master packetiser transform packs compressed audio frames (SBC, AAC, aptX) from the audio subsystem into TWS packets for transmission over the air to the TWS slave. The TWS slave packetiser transform receives TWS packets over the air from the TWS master. It unpacks compressed audio frames and writes them to the audio subsystem.

Source prompt_source

The prompt file source whilst prompt is playing

kymera_chain_handle_t sco_handle
