Detailed Description

Indication message for an incoming OBEX PUT request.

On receiving this indication message, the application must call ObexPutResponse() with the response code. If the application wants to add any Object headers, it must add it by creating new Object using ObexObJNew() and adding the headers using ObexObjAddByteHdr() or ObexObjAddVariableHdr() API.

The application may use ObexObjMapHdrValue() or ObexObjFilterHdrValue() or ObexObjExtractByteHdrValue() API to extract the headers from the Source buffer. The application must not access the source buffer after calling ObexPutResponse(). The Source buffer will be deleted for receiving the next OBEX request packet after sending the response.

The application may treat a Source object without any Body/End-Of-Body headers as a Delete request and a Source Object with empty End-of-Body header as a Create request.
